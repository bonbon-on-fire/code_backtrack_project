# v2.5 Manual Verification — overtype / cut

Step 4 of the v2.5 build order: run each behavior in a **real editor** and diff
expected vs. actual counts. This is the one step the automated tests can't cover
(no real keyboard hook), so it's done by hand.

## How to run
1. Start the tracker: `python -m backspace_tracker`
2. For **each** numbered row below:
   - Do any **Setup** (type seed text / fill clipboard) *before* recording.
   - Press **Ctrl+Alt+B** to start a fresh session.
   - Press **exactly** the keys in the row — nothing else (every stray key
     lands in `OTHER`; clicking with the mouse is fine, it isn't hooked).
   - Press **Ctrl+Alt+B** to stop; answer `n` at the save prompt (no need to keep these).
   - Read the summary and compare to **Expected**. Record the actual in the table.
3. Categories: BS=Backspace, C-BS=Ctrl+Backspace, DEL=Delete, C-DEL=Ctrl+Delete,
   UNDO=Ctrl+Z, OVR=Overtype, CUT=Ctrl+X, OTH=Other. Reminder: OVR/CUT show in the
   **summary only**, not the live line.

## Known blind spots (expected to MISS — confirm the gap, don't "fix")

These are accepted limits (PLANNING.md → Known Limits). The point is to *confirm*
the tracker quietly under-counts here, so the data's caveats are documented.

| # | Behavior | Keys / action | Expected (the gap) |
|---|----------|---------------|--------------------|
| B1 | Mouse-drag select + type-over | drag-select a word, type `x` | OVR=0, OTH=1 — no mouse hook |
| B2 | Double-click word + type-over | double-click a word, type `y` | OVR=0, OTH=1 — no mouse hook |
| B3 | Autocomplete accept | type `fun`, accept completion → `function` | only the typed keys + accept key counted; replacement invisible |
| B4 | Multi-cursor magnitude | 3 cursors, press Backspace **once** | BS=1 (not 3) — magnitude lost |
| B5 | Vim normal-mode delete (vim users) | `dd` or `ciw` in normal mode | OTH only; deletion invisible |
